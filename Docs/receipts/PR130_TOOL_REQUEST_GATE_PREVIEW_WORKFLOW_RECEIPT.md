PR130 adds a Tool Request and Gate Preview candidate workflow. It turns supplied evidence and proposal material into a safe tool request preview package.

Tool request preview is not tool execution.

Gate preview is not gate satisfaction.

Capability label is not authorization.

Expected output reference is not produced output.

Evidence is not approval.

It does not invoke tools, authorize tools, reserve tools, run commands, call models, build prompts, dispatch agents, satisfy approval, satisfy policy, transition workflow state, mutate source, apply patches, create tickets, promote memory, activate retrieval, or add runtime wiring.

Preview output cannot grant authority.

This is a Block M L4 candidate workflow and remains non-mutating.

The workflow operates only on supplied safe evidence and snapshots. It does not query live tool registries, call GitHub, inspect CI, read repository files, read logs from disk, create SQL records, or expose API/CLI/UI/runtime hosting.

Review line: PR130 previews the locked tool gate. It does not open it.
