# Refactor Candidate Inventory

**Status:** Supporting refactor inventory

**Snapshot:** 15 July 2026

**Programme slice:** CLN-34

**Selection rule:** tracked executable service, controller, hook, or component at roughly 800 lines or more on the cumulative CLN-33 head; generated output, test harnesses, command hosts, and model-only contract files are recorded separately, not silently treated as split candidates.

Size is a triage signal, not permission to refactor. Authority-sensitive classes must keep their current gates, evidence requirements, project boundaries, and failure behavior while seams are introduced. Characterization precedes extraction.

| `IronDev.TauriShell/src/api/ironDevApi.ts` | Hand-written API facade; it is genuine executable debt, but decomposition belongs to a dedicated API-client contract slice with drift coverage. |
| `IronDev.Core/*Models.cs` files above the threshold | Dense immutable contract families. Split only when namespaces and serialization compatibility have an independently justified boundary. |
| `IronDev.Core/Governance/FrontendReadinessReadModels.cs` | API read-model contract surface; size alone does not justify reshaping serialized truth. |
| Integration and evaluation files above the threshold | Test/evaluation harness size is tracked separately; production refactor entry criteria do not authorize test-suite rewrites. |
| `tools/IronDev.Cli/*` and `tools/IronDev.ReplayRunner/*` command hosts above the threshold | Large executable tooling is real debt, but it requires a separate command-boundary inventory and cannot be inferred from service seams. |

## Entry criteria for a refactor PR

1. Name one responsibility and one seam from this inventory.
2. Add characterization for success, refusal/failure, missing scope, and stale/partial evidence where relevant.
3. Keep public routes, DTOs, generated clients, DI lifetimes, audit fields, and authority decisions unchanged unless the PR is explicitly a contract change.
4. Prove the old and new path cannot both own the same truth.
5. Stop if extraction requires inventing a default, permission, approval, or fallback result.
