# Governed Planner/Critic Tool Loop 162-167

IRONDEV-162 through IRONDEV-167 add the first native governed reasoning loop.

The loop lets PlannerAgent request evidence through named capabilities, lets the tool registry validate and execute safe read/test/report actions, lets CriticAgent review the collected evidence, lets PlannerAgent revise the plan, and records a human escalation decision.

Agents request capabilities such as `memory.search`, `code.search`, `trace.read`, `failure.latest`, `test.run-plan`, `quality.run-gate`, and `project.build`. They do not execute raw shell commands directly.

The loop introduces a language-agnostic runtime profile contract. .NET is the first supported adapter, but Node and Python profiles are also represented so future projects do not have to pretend they are .NET projects.

The evidence validation layer checks whether required evidence is present before a revised plan can be trusted. The human escalation gate decides whether the result is ready for review, needs more evidence, or must be blocked.

This checkpoint does not grant real repository writes, memory mutation, ticket creation, patch application, raw command execution by agents, ConscienceAgent bypass, ThoughtLedger bypass, or self-approval.
